## 澄清会话 2026-04-13

### Q1：行动判定的执行顺序
**答：C - AI综合判断（场景合理性优先）**

AI判定行动时，根据每个角色当前的位置、时间、处境等场景逻辑决定顺序，不以力量高低或提交先后为准。

> 例：A角色在目标房间内，B角色在两里外的山上——A的行动先发生，无论谁先提交。

---

### Q2：超出角色能力的行动如何处理
**答：D - 提交时前置警告 + AI按真实规则判定**

- 网站提交界面显示该行动的**预估成功概率**
- 玩家确认后，AI仍按世界规则判定（可能重伤/失败/被嘲讽）
- 不做任何"保护性降级"处理

> 效果：信息透明 + 规则严肃。玩家冒险是知情的，后果是真实的。

### Q3：玩家个人角色日报系统
**答：双轨叙事系统（重大设计决策）**

```
轨道①：官方主线小说（起点/番茄，所有读者共享）
  → 纯主角视角，记录当前主角的所见所行

轨道②：个人角色日志（网站，用户私人空间）
  → 以用户自己角色为视角的每日小故事
  → AI自主根据用户目标 + 世界规则 + 当日事件，生成该角色一天的经历
  → 包含：叙事性小说 + 修炼数值变化
  → 用户体验：窥视感（看AI帮角色做决定）+ 参与感（目标是我设的）
```

这是产品最核心的差异化体验：用户不是在"玩游戏"，而是在"养一个活在另一个世界里的人"。

### Q4：用户参与度光谱设计
**答：参与度从放养到深度，需要结构引导**

```
放养模式（轻度用户）：
  · 只设定一个长期目标（如：成为内门弟子）
  · AI完全自主安排角色每天的行动
  · 用户只需每天打开看日志即可

深度模式（核心用户）：
  · 可指定详细的每日/每小时方向指令
  · 但必须有：
      1. 任务指引（AI根据角色当前状态推荐今日优先行动）
      2. 全服事件公告（世界当前发生的事，让玩家知道今天有什么可以参与）
      3. 其他玩家的公开动态（排行榜、世界公开事件日志）
```

> 关键原则：自由度越高，结构支撑越重要。无引导的高自由度 = 玩家流失。

### Q5：世界事件的触发机制
**答：世界中本就存在，按内在逻辑自然浮现（小说思维）**

> "按照故事大纲或者世界观设定，本来就存在世界中的，不是后期随机加进去的。跟其他小说一样，一开始没说不代表没有。"

**核心哲学**：
- 荒灵原的上古遗迹从第一天就在那里，只是还没到被发现的时机
- 宗门大比每90天就会发生，从天机宗建立那天起就是规则
- 魔族封印波动有其节律，不需要任何人"触发"

**三类事件触发机制：**

| 类型 | 触发条件 | 示例 |
|------|---------|------|
| **世界日历型** | 时间到了自动发生 | 月度大比、妖兽活跃期、宗门纪念日 |
| **玩家行为阈值型** | 足够多的玩家聚集在某区域，"发现"了本就存在的东西 | 大量玩家探索荒灵原 → 遗迹被"发现" |
| **主线剧情联动型** | 官方小说叙事推进到某个节点，世界相应解锁 | 主角章节提到"北荒异动" → 全服感知到封印波动 |

**玩家如何知道有事件可参与**：
- AI根据世界当前状态，在任务指引中写入"近期有传言荒灵原深处出现异动"
- 宗门公告栏（网站内）同步世界公开信息
- 其他玩家的公开行动轨迹形成的"热点区域"提示
